TRIPLE - O


Objective - Design of a Private cloud in the campus using Triple O(OpenStack on OpenStack ) A private cloud, also called as an "internal cloud" resides within the organizational boundary. The access to this internal cloud is restricted to internal users according to their role.


Overview - TripleO (OpenStack On OpenStack) is a program aimed at installing, upgrading and operating OpenStack clouds using OpenStack's own cloud facilities as the foundations - building on nova, neutron and heat to automate fleet management at datacentre scale.


Methods - Cluster the server (Linux/RedHat),
Install the OpenStack on the server and then virtualize the instance and install the second instance of OpenStack  over the server 


Outcomes/Requirements


Intended outcome :
Able to create our internal cloud in the campus and access according to our own privileges
